## 免费部署方案

### 方案一：GitHub Pages（最推荐新手）

#### 🎯 优势
- ✅ 完全免费
- ✅ 自动HTTPS
- ✅ 全球CDN加速
- ✅ 支持自定义域名
- ✅ 版本控制集成

#### 📝 部署步骤

**第一步：创建GitHub账户**
1. 访问 https://github.com
2. 点击右上角 "Sign up"
3. 填写用户名、邮箱、密码
4. 验证邮箱账户

**第二步：创建新仓库**
1. 登录后点击右上角 "+" → "New repository"
2. 仓库名称填写：`portfolio-website`
3. 选择 "Public"（公开）
4. 勾选 "Add a README file"
5. 点击 "Create repository"

**第三步：准备文件**

**方式一：直接使用现有文件（推荐）**
```
/Users/yangjie/Desktop/1/1236/
├── index-animated.html
└── .figma/
    └── image/
        ├── mn4f6hg3-c53ddxc.svg
        ├── mn4f6hg3-aud1943.svg
        ├── mn4f6hg8-i7n4lel.png
        └── ... (所有其他图片文件)
```

**方式二：创建部署文件夹（可选）**
如果需要创建专门的部署文件夹，可以这样做：
```
portfolio-website/
├── index-animated.html
└── .figma/
    └── image/
        ├── mn4f6hg3-c53ddxc.svg
        ├── mn4f6hg3-aud1943.svg
        ├── mn4f6hg8-i7n4lel.png
        └── ... (所有其他图片文件)
```

**推荐使用方式一**，直接使用现有的文件结构。

**第四步：上传文件到GitHub**

**方法A：使用网页界面（推荐新手）**
1. 进入刚创建的仓库
2. 点击 "Add file" → "Upload files"
3. 拖拽整个文件夹到上传区域
4. 或点击 "choose your files" 选择文件
5. 在 "Commit changes" 输入：`Initial commit`
6. 点击 "Commit changes"

**方法B：使用Git命令（适合进阶）**

**场景一：直接使用现有文件（推荐）**
```bash
# 1. 进入现有项目目录
cd /Users/yangjie/Desktop/1/1236

# 2. 初始化Git仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交更改
git commit -m "Initial commit"

# 5. 添加远程仓库（替换YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/portfolio-website.git

# 6. 推送到GitHub
git branch -M main
git push -u origin main
```

**场景二：创建专门的部署文件夹**
```bash
# 1. 在本地创建文件夹
mkdir portfolio-website
cd portfolio-website

# 2. 复制文件
cp -r /Users/yangjie/Desktop/1/1236/index-animated.html .
cp -r /Users/yangjie/Desktop/1/1236/.figma .

# 3. 初始化Git仓库
git init

# 4. 添加所有文件
git add .

# 5. 提交更改
git commit -m "Initial commit"

# 6. 添加远程仓库（替换YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/portfolio-website.git

# 7. 推送到GitHub
git branch -M main
git push -u origin main
```

**第五步：启用GitHub Pages**
1. 进入仓库页面
2. 点击 "Settings" 标签页
3. 滚动到 "Pages" 部分
4. 在 "Build and deployment" 下：
   - Source 选择：`Deploy from a branch`
   - Branch 选择：`main`
5. 点击 "Save"

**第六步：等待部署**
- ⏱️ 等待1-3分钟
- ✅ 部署完成后会显示绿色勾选
- 🌐 访问地址：`https://YOUR_USERNAME.github.io/portfolio-website/`

#### 🔗 自定义域名（可选）
1. 在仓库根目录创建 `CNAME` 文件
2. 文件内容填写您的域名：`www.yourdomain.com`
3. 保存并提交到GitHub
4. 在域名DNS设置中添加CNAME记录指向 `YOUR_USERNAME.github.io`

### 方案二：Vercel（推荐快速部署）

#### 🎯 优势
- ✅ 完全免费
- ✅ 自动HTTPS
- ✅ 极速全球CDN
- ✅ 自动部署
- ✅ 支持自定义域名

#### 📝 部署步骤

**第一步：注册Vercel账户**
1. 访问 https://vercel.com
2. 点击 "Sign Up"
3. 使用GitHub账户登录（推荐）

**第二步：导入项目**
1. 点击 "Add New Project"
2. 选择 "Continue with GitHub"
3. 授权Vercel访问您的GitHub
4. 选择 `portfolio-website` 仓库
5. 点击 "Import"

**第三步：配置项目**
1. Project Name：`portfolio-website`
2. Framework Preset：`Other`
3. Root Directory：留空或 `./`
4. 点击 "Deploy"

**第四步：等待部署**
- ⏱️ 等待30秒-2分钟
- ✅ 部署完成后会显示绿色勾选
- 🌐 访问地址：`https://portfolio-website.vercel.app`

#### 🔗 自定义域名（可选）
1. 进入项目 "Settings" → "Domains"
2. 点击 "Add Domain"
3. 输入您的域名：`www.yourdomain.com`
4. 按照提示配置DNS记录

## 付费部署方案

### 方案四：传统虚拟主机（适合完全控制）

#### 🎯 优势
- ✅ 完全控制
- ✅ 支持多种技术
- ✅ 数据库支持
- ✅ 邮箱服务

#### 📝 部署步骤

**第一步：购买虚拟主机**
推荐服务商：
- 阿里云：https://www.aliyun.com（国内推荐）
- 腾讯云：https://cloud.tencent.com
- Bluehost：https://www.bluehost.com（国外推荐）
- Hostinger：https://www.hostinger.com（便宜推荐）

**第二步：购买域名**
- 可在主机商处一起购买
- 或单独购买：阿里云、腾讯云、Namecheap

**第三步：配置主机环境**
1. 登录主机控制面板
2. 创建新网站/站点
3. 记录以下信息：
   - FTP主机地址
   - FTP用户名
   - FTP密码
   - 网站根目录

**第四步：使用FTP上传文件**

**使用FileZilla（Windows/Mac）**
1. 下载安装：https://filezilla-project.org/download.php
2. 打开FileZilla
3. 点击 "文件" → "站点管理器"
4. 填写信息：
   - 主机：您的FTP地址
   - 用户：FTP用户名
   - 密码：FTP密码
   - 端口：21（默认）
5. 点击 "快速连接"
6. 连接成功后，左侧显示远程文件
7. 右侧显示本地文件
8. 拖拽 `portfolio-website` 文件夹到 `public_html` 或 `www` 目录
9. 等待上传完成

**使用WinSCP（Windows）**
1. 下载安装：https://winscp.net/
2. 新建站点：主机名、用户名、密码
3. 连接后显示左右文件面板
4. 从左侧本地选择文件，拖拽到右侧远程目录
5. 等待上传完成

**第五步：访问网站**
- 🌐 输入您的域名：`www.yourdomain.com`
- ✅ 网站应该正常显示

## 域名配置

### 🌐 域名购买推荐

| 域名商 | 价格 | 优势 | 推荐度 |
|--------|------|------|----------|
| 阿里云 | 中等 | 国内访问快 | ⭐⭐⭐⭐⭐⭐ |
| 腾讯云 | 中等 | 国内访问快 | ⭐⭐⭐⭐⭐ |
| Namecheap | 便宜 | 国际知名 | ⭐⭐⭐⭐⭐ |
| GoDaddy | 贵 | 服务好 | ⭐⭐⭐ |

### 📝 DNS配置步骤

**阿里云域名配置**
1. 登录阿里云控制台
2. 进入 "域名" → "域名列表"
3. 点击您的域名 "解析"
4. 添加记录：
   - 记录类型：`CNAME`
   - 主机记录：`www`
   - 记录值：`YOUR_USERNAME.github.io`（GitHub Pages）
   - 或：`portfolio-website.vercel.app`（Vercel）
   - 或：`portfolio-website.netlify.app`（Netlify）
5. TTL：600（默认）
6. 点击 "确认"

**腾讯云域名配置**
1. 登录腾讯云控制台
2. 进入 "域名" → "DNS解析"
3. 点击 "添加记录"
4. 选择记录类型：`CNAME`
5. 主机记录：`www`
6. 记录值：填写您的部署地址
7. TTL：600
8. 点击 "保存"

**DNS生效时间**
- ⏱️ 通常需要10分钟-24小时
- ✅ 可使用 `nslookup` 命令检查：`nslookup www.yourdomain.com`

## 常见问题

### ❓ 部署失败怎么办？

**GitHub Pages部署失败**
1. 检查仓库名称是否正确
2. 确保分支是 `main` 而不是 `master`
3. 检查Settings中Pages是否启用
4. 查看Actions标签页是否有部署错误

**Vercel部署失败**
1. 检查项目配置是否正确
2. 查看Deployment日志
3. 确保文件路径正确
4. 重新触发部署

**Netlify部署失败**
1. 检查文件结构
2. 确保index.html在根目录
3. 查看Deploy日志
4. 尝试重新部署

### ❓ 图片不显示怎么办？

**检查路径问题**
1. 确认图片路径：`.figma/image/文件名`
2. 检查文件名大小写是否正确
3. 确保图片文件已上传
4. 使用浏览器开发者工具检查404错误

**解决方案**
- GitHub：确保图片在仓库的 `.figma/image/` 文件夹中
- 虚拟主机：确保上传了整个 `.figma` 文件夹
- 检查相对路径是否正确

### ❓ 如何更新网站？

**GitHub Pages**
1. 修改本地文件
2. 使用Git提交：`git add . && git commit -m "Update" && git push`
3. 自动触发重新部署
4. 等待1-3分钟

**Vercel/Netlify**
1. 修改本地文件
2. 推送到GitHub（自动触发部署）
3. 或直接在平台重新部署

**虚拟主机**
1. 修改本地文件
2. 使用FTP重新上传修改的文件
3. 覆盖原文件即可

### ❓ HTTPS证书问题？

**免费方案**
- ✅ GitHub/Vercel/Netlify 自动提供HTTPS
- ✅ 无需额外配置

**虚拟主机**
1. 大多数主机商提供免费Let's Encrypt证书
2. 在控制面板启用SSL
3. 强制HTTPS重定向
